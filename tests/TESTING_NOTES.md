# Testing Notes

Generated by Testing Workbench.

Suggested flow:

1. Run the generated smoke tests first.
2. Run the workbench button inventory and callback audit.
3. Open `gui_action_recorder.py` for manual button-by-button checks.
4. Save logs after each meaningful run so changes can be compared later.

The recorder helper is intentionally manual. It is safest for desktop apps
because it lets you verify what each button does without forcing the target
application into a fragile automated click script.
